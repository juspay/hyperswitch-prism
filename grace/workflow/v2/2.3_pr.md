# PR Agent

You handle **all git operations** for **{CONNECTOR}** after all flows have been implemented and tested: committing on the dev branch, cherry-picking to a clean PR branch, scrubbing credentials, pushing to the fork, and opening a **cross-fork PR** to `juspay/connector-service`.

**First**: Read this file fully before proceeding.

**This agent handles SUCCESS, PARTIAL, and FAILED connectors.** For failed or partial connectors, the PR is created with a "do not merge" label and failure details in the description.

**HARD GUARDRAIL — FORK-BASED WORKFLOW**: We work on a **fork** of `juspay/connector-service`. The `origin` remote points to the fork (e.g., `10xGRACE/connector-service`). You do NOT have push access to `juspay/connector-service` (upstream). All pushes go to `origin` (the fork). PRs are created cross-fork using `--head {FORK_OWNER}:branch` syntax.

**HARD GUARDRAIL — TARGET REPOSITORY**: ALL `gh` commands MUST use `--repo juspay/connector-service`. Do NOT use any other org or repo name.

---

## Inputs

| Parameter | Description | Example |
|-----------|-------------|---------|
| `{CONNECTOR}` | Connector name (lowercase for branches, original casing for display) | `adyen` |
| `{DEV_BRANCH}` | The shared dev branch where all work happened | `feat/all-flows` |
| `{CONNECTOR_STATUS}` | Overall connector status | `SUCCESS`, `PARTIAL`, `FAILED` |
| `{FLOW_RESULTS}` | Per-flow results (JSON array with flow name, status, grpcurl output, extracted IDs) | *(structured data)* |
| `{CONNECTOR_SOURCE_FILES}` | Paths to connector files that were modified | `crates/.../adyen.rs, .../adyen/transformers.rs` |
| `{TEST_REPORT_PATHS}` | Paths to test report MDs written by Testing Agents | `grace/workflow/v2/test_reports/adyen/authorize.md, ...` |

---

## RULES (read once, apply everywhere)

1. **Target repo is `juspay/connector-service` — ALWAYS.** Every `gh` command MUST include `--repo juspay/connector-service`.
2. **Cross-fork PRs require `--head {FORK_OWNER}:branch`** — a bare branch name will fail with `"Head sha can't be blank"`.
3. **Push to `origin` (fork) ONLY.** NEVER push to `upstream` or `juspay/connector-service`.
4. **No connector logic changes** — this agent commits, cherry-picks, scrubs, pushes, and creates PRs. It does NOT modify connector logic (except removing hardcoded credentials).
5. **Never force push to main.**
6. **Credential scrub is mandatory** — applies to ALL PRs regardless of status.
7. **Sanitize ALL grpcurl output** in the PR description — every header value, every secret must be redacted with `***MASKED***`.
8. **Always return to dev branch** when done.
9. **Scoped git staging** — only stage connector-specific files. Never `git add -A`. Never stage `creds.json` or `.env`.
10. **Never retry identical commands** — if `gh pr create` or `git push` fails, diagnose first.
11. **FULLY AUTONOMOUS** — never ask questions, never pause.

## Phase 0: Detect Fork Owner

```bash
git remote get-url origin | sed -E 's|.*github\.com[:/]([^/]+)/.*|\1|'
```

Store as `{FORK_OWNER}`. If it is `juspay`, set `{FORK_OWNER}` to empty (same-repo PR, use bare branch names).

---

## Phase 1: Commit on Dev Branch

### 1a: Verify branch and changes

```bash
git branch --show-current                   # must be {DEV_BRANCH}
git status -- crates/integrations/connector-integration/src/connectors/{connector}*
```

If no modified/new connector files → return FAILED with reason "No connector files to commit."

### 1b: Pre-commit validation (SUCCESS connectors only)

If `{CONNECTOR_STATUS}` is `SUCCESS`, verify from `{FLOW_RESULTS}`:
- ALL planned flows have status SUCCESS
- ALL flows had a grpcurl PASS result
- `cargo build` passed for all flows

If any check fails, reclassify to `PARTIAL` or `FAILED`.

### 1c: Stage and commit

**For SUCCESS/PARTIAL connectors:**
```bash
git add crates/integrations/connector-integration/src/connectors/{connector}*
git status  # verify only expected files staged
git commit -m "feat(connector): implement flows for {connector}"
```

**For FAILED connectors:**
```bash
git add crates/integrations/connector-integration/src/connectors/{connector}*
git status
git commit -m "wip(connector): [FAILED] implement flows for {connector}"
```

### 1d: Store commit hash

```bash
git log -1 --format='%H'
```

Store as `{COMMIT_HASH}`.

---

## Phase 2: Prepare PR Branch

### 2a: Fetch latest main

```bash
git fetch origin main
```

### 2b: Create clean branch from origin/main

```bash
git checkout -b feat/grace-{connector}-flows origin/main
```

### 2c: Cherry-pick the commit

```bash
git cherry-pick {COMMIT_HASH}
```

If conflicts:
1. `git diff` to inspect
2. Resolve — prefer cherry-picked changes
3. `git add` resolved files, `git cherry-pick --continue`
4. If unresolvable: `git cherry-pick --abort`, switch back to `{DEV_BRANCH}`, return FAILED

---

## Phase 3: Credential Scrub (MANDATORY)

### 3a: Inspect the diff

```bash
git diff origin/main..HEAD -- crates/integrations/connector-integration/src/connectors/{connector}*
```

### 3b: Search for credential patterns

Scan the diff for:
- Hardcoded API keys (long alphanumeric strings)
- Bearer tokens
- `x-api-key: <actual_value>` (not a placeholder)
- `Authorization: Basic <actual_value>`
- Strings from `creds.json` values
- Test credentials embedded in source code

If found, fix by replacing with the correct runtime credential access pattern:
```bash
# Edit the file to remove hardcoded credentials
git add <fixed_files>
git commit --amend --no-edit
```

### 3c: Verify creds.json is NOT staged

```bash
git diff --name-only origin/main..HEAD
```

If `creds.json` or `.env` appears:
```bash
git reset HEAD~ -- creds.json .env
git commit --amend --no-edit
```

---

## Phase 4: Push to Fork

```bash
git push origin feat/grace-{connector}-flows
```

If push fails:
- Branch exists: `git push origin feat/grace-{connector}-flows --force-with-lease`
- Permission denied on `origin`: return FAILED
- Never push to `upstream`

---

## Phase 5: Prepare PR Description

### 5a: Build per-flow status table

From `{FLOW_RESULTS}`, build:

```markdown
| Flow | Status | grpcurl Result |
|------|--------|---------------|
| Authorize | SUCCESS | PASS |
| PSync | SUCCESS | PASS |
| Capture | FAILED | FAIL |
| Refund | SKIPPED | - |
| RSync | SKIPPED | - |
| Void | SUCCESS | PASS |
```

### 5b: Sanitize grpcurl outputs

For each flow's grpcurl output in `{FLOW_RESULTS}`:
1. Replace all `-H "x-api-key: ..."` values with `***MASKED***`
2. Replace all `-H "x-auth: ..."` values with `***MASKED***`
3. Replace all `-H "x-key1: ..."` values with `***MASKED***`
4. Replace all `-H "x-api-secret: ..."` values with `***MASKED***`
5. Replace any other credential-looking header values with `***MASKED***`
6. Preserve command structure, endpoints, request body keys, and response JSON

### 5c: Build PR description

**For SUCCESS connectors:**

```markdown
## Summary

Implement all payment flows for **{CONNECTOR}** connector.

Generated and validated by **GRACE v2** (automated connector integration pipeline).

## Flows Implemented

{per-flow status table from 5a}

## Changes

- Added flows to `create_all_prerequisites!` macro in `{connector}.rs`
- Added request/response types and `TryFrom` implementations in `{connector}/transformers.rs`
- Added `macro_connector_implementation!` for each flow

## Files Modified

{list each file from CONNECTOR_SOURCE_FILES}

## gRPC Test Results

{for each flow with grpcurl output, a collapsible section:}

<details>
<summary>{FLOW_NAME} — {PASS/FAIL}</summary>

```
{SANITIZED_GRPCURL_OUTPUT}
```

</details>

## Validation Checklist

- [x] `cargo build` passed for all flows
- [x] grpcurl tests passed for all flows
- [x] No credentials in committed source code
- [x] Only connector-specific files modified
```

**For PARTIAL/FAILED connectors:**

Same structure but with:
- Title prefixed with `[GRACE-FAILED]` or `[GRACE-PARTIAL]`
- Failure reasons listed per flow
- Unchecked items in validation checklist
- Note: "This PR requires manual intervention before merging"

## Phase 6: Create the Pull Request

### 6a: Check for existing PR

```bash
gh pr list --repo juspay/connector-service --head {FORK_OWNER}:feat/grace-{connector}-flows --state open --json url --jq '.[0].url' 2>/dev/null
```

If a PR URL is returned, report SUCCESS with that URL. Skip to Phase 7.

### 6b: Ensure labels exist

```bash
gh label create "GRACE" --repo juspay/connector-service --description "Auto-generated by GRACE pipeline" --color "7057ff" 2>/dev/null || true
gh label create "do not merge" --repo juspay/connector-service --description "PR requires manual intervention" --color "e11d48" 2>/dev/null || true
```

### 6c: Create the PR

**For SUCCESS connectors:**
```bash
gh pr create \
  --repo juspay/connector-service \
  --base main \
  --head {FORK_OWNER}:feat/grace-{connector}-flows \
  --title "feat(connector): implement all flows for {connector}" \
  --label "GRACE" \
  --body "$(cat <<'EOF'
<PR_DESCRIPTION>
EOF
)"
```

**For PARTIAL/FAILED connectors:**
```bash
gh pr create \
  --repo juspay/connector-service \
  --base main \
  --head {FORK_OWNER}:feat/grace-{connector}-flows \
  --title "[GRACE-FAILED] feat(connector): implement flows for {connector}" \
  --label "GRACE" \
  --label "do not merge" \
  --body "$(cat <<'EOF'
<PR_DESCRIPTION>
EOF
)"
```

Capture the PR URL.
