## Implementation Patterns

All scenarios follow the same 4-step pattern:

```
1. CONFIGURE  → 2. AUTHORIZE  → 3. HANDLE STATUS  → 4. CAPTURE/SETTLE
```

### Step 1: Configure
```python
# Universal pattern - only change connector name
config = ConnectorConfig(
    connector="stripe",  # ← Change this
    environment=Environment.SANDBOX,
    auth=load_credentials("stripe"),  # Connector-specific
)
```

### Step 2: Authorize
```python
# Universal request structure across all connectors
response = client.authorize(
    merchant_transaction_id="txn_001",
    amount={"minor_amount": 1000, "currency": "USD"},
    payment_method={"card": {...}},
    capture_method="MANUAL",
)
```

### Step 3: Handle Status
```python
if response.status == "AUTHORIZED":
    # Proceed to capture
elif response.status == "PENDING":
    # Poll or wait for webhook
elif response.status == "FAILED":
    # Show error
```

### Step 4: Capture/Settle
```python
client.capture(
    connector_transaction_id=response.connector_transaction_id,
    amount_to_capture={"minor_amount": 1000, "currency": "USD"},
)
```

## Running Examples

### Python
```bash
# Card payment with any connector
python examples/scenarios/checkout_card.py --connector=stripe

# Wallet payment
python examples/scenarios/checkout_wallet.py --connector=adyen

# Bank transfer
python examples/scenarios/checkout_bank.py --connector=stripe --method=sepa
```

### Rust
```bash
# Card payment with any connector
cargo run --example checkout_card -- --connector=stripe

# With credentials
cargo run --example checkout_card -- --connector=adyen --credentials=creds.json
```

### All Scenarios
```bash
# List all available scenarios
ls examples/scenarios/

# Run with dry-run (no API calls)
python examples/scenarios/checkout_card.py --connector=stripe --dry-run
```
